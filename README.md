## The Problem

Sales reps run **dozens of booth demos per day** at trade shows. After the event,
they remember almost nothing -- who was interested in what, which features clicked,
what follow-up was promised. Leads go cold. Revenue is left on the table.

## The Solution

BoothApp captures the **full context** of every demo -- screen activity, audio,
and visitor badge -- then uses AI to generate a structured report with engagement
scores, topic detection, and recommended next steps. All automated, all in real time.

## Quick Start

```bash
# 1. Install dependencies
npm install

# 2. Configure AWS credentials
export AWS_PROFILE=hackathon

# 3. Run preflight check (validates AWS, Node, bucket access)
bash scripts/preflight.sh

# 4. Start the watcher
npm run watcher
```

The watcher polls S3 for new session uploads, runs the analysis pipeline
(transcription -> correlation -> AI analysis), and writes an HTML report
to each session's output directory.

---

## How It Works

1. **Capture** -- The Chrome extension records screen activity and audio
   while the badge scanner associates the session with a visitor.

2. **Upload** -- Session data (audio, click events, screenshots) is uploaded
   to S3 via pre-signed URLs generated by a Lambda function.

3. **Analyze** -- The watcher detects new sessions and runs a three-stage
   pipeline: transcription, event correlation (matching clicks to transcript
   segments and screenshots within a 2-second window), and AI analysis via
   Amazon Bedrock.

4. **Report** -- An HTML report is generated with engagement scores,
   detected product topics (XDR, Endpoint Security, ZTSA, Cloud Security,
   Email Security), and AI-recommended follow-up actions.

## Tech Stack

| Layer | Technology |

| Storage | Amazon S3 |
