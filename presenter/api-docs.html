GET /api/health System health & uptime
Description

Returns server health status, uptime in seconds, active session count, and version.

Response
{
  "status": "ok",
  "uptime": 3472,
  "sessions_count": 0,
  "version": "1.0.0"
}
Example
curl http://localhost:3000/api/health
GET /api/config Server configuration
Description

Returns current server configuration: S3 bucket, AWS region, port, and Node.js version.

Response
{
  "s3_bucket": "boothapp-sessions-752266476357",
  "aws_region": "us-east-1",
  "port": 3000,
  "node_version": "v20.11.0"
}
Example
curl http://localhost:3000/api/config
GET /api/pages List available HTML pages
Description

Returns a list of all HTML page files served by the presenter.

Response
{
  "pages": [
    { "name": "admin", "path": "/admin.html" },
    { "name": "sessions", "path": "/sessions.html" },
    { "name": "api-docs", "path": "/api-docs.html" }
  ]
}
Example
curl http://localhost:3000/api/pages
GET /api/session/:id/screenshots Session screenshots & click data
Description

Lists all screenshots captured during a session with presigned S3 URLs (valid 1 hour). Also returns click event data if available.

URL Parameters
NameTypeDescription
id string Session ID (alphanumeric, hyphens, underscores)
Response
{
  "session_id": "demo-2026-03-31",
  "screenshots": [
    {
      "key": "sessions/demo-2026-03-31/screenshots/001.png",
      "filename": "001.png",
      "url": "https://s3.amazonaws.com/...",
      "size": 245832,
      "last_modified": "2026-03-31T12:00:00Z"
    }
  ],
  "clicks": [
    {
      "url": "https://portal.trendmicro.com/dashboard",
      "x": 450,
      "y": 320,
      "timestamp": 1711872000000
    }
  ]
}
Error Responses
StatusCondition
400Invalid session ID format
404Session not found in S3
Example
curl http://localhost:3000/api/session/demo-2026-03-31/screenshots
Try it
POST /api/analyze-batch Queue sessions for batch AI analysis
Description

Submits one or more sessions for AI analysis via Claude/Bedrock. Sessions are processed sequentially. Maximum 100 sessions per batch. Previously analyzed sessions are re-analyzed.

Request Body
FieldTypeDescription
session_ids string[] Array of session IDs to analyze (1-100) 
{
  "session_ids": ["session-001", "session-002"]
}
Response
{
  "batch_id": "m1abc2d",
  "queued": 2,
  "already_analyzed": 1,
  "total": 2
}
Error Responses
StatusCondition
400session_ids missing or empty
400More than 100 sessions
500S3 connectivity error
Example
curl -X POST http://localhost:3000/api/analyze-batch \
  -H "Content-Type: application/json" \
  -d '{"session_ids": ["session-001", "session-002"]}'
Try it
GET /api/analyze-status Batch analysis progress
Description

Returns the current batch analysis progress: queued, running, completed, and failed counts with percentage. Poll this endpoint after submitting a batch.

Response (active batch)
{
  "active": true,
  "batch_id": "m1abc2d",
  "started_at": "2026-03-31T12:00:00.000Z",
  "total": 5,
  "queued": 2,
  "running": 1,
  "completed": 2,
  "failed": 0,
  "progress_pct": 40,
  "errors": []
}
Response (no batch)
{
  "active": false,
  "message": "No batch has been submitted"
}
Example
curl http://localhost:3000/api/analyze-status
